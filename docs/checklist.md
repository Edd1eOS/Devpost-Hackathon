# Build Checklist — AI Audit Assistant

## Build Preferences

- **Build mode:** Step-by-step
- **Comprehension checks:** Yes — after each item, ask one short question to confirm the module clicked
- **Git:** Commit after each item with message: `"Complete step N: [title]"`
- **Verification:** Yes — after each item, run the app or module directly and confirm expected output before moving on
- **Check-in cadence:** Learning-driven — after each item, briefly explain: (1) what files changed, (2) what the module does, (3) how to test it. Then comprehension check question.

- [ ] **4. Receipt Matcher + OCR Extraction**
  Spec ref: `spec.md > Engine Layer > engine/matcher.py`, `spec.md > Engine Layer > engine/ocr.py`
  What to build: (1) `src/engine/matcher.py` — implement `match_receipts(df: pd.DataFrame, uploaded_files: dict[str, bytes]) -> tuple[dict[str, bytes | None], list[str]]`. Maps each `receipt_file` value in the DataFrame to its image bytes from `uploaded_files`; produces `None` for filenames not found; returns `(matched_dict, missing_list)`. (2) `src/engine/ocr.py` — implement `extract(image_bytes: bytes) -> OCRResult`. Open image via `PIL.Image.open(BytesIO(image_bytes))`. Run `pytesseract.image_to_data()` with `TESSERACT_CONFIG = "--psm 6"` to get per-word confidence; compute `mean(confidences) / 100`. Run `pytesseract.image_to_string()` for raw text. Apply regex to extract amount (`r'\$?\d{1,6}[\.,]\d{2}'`), date (`r'\d{1,2}[\/\-]\d{1,2}[\/\-]\d{2,4}'`), vendor (first non-numeric line). Return `OCRResult`. Set `CONFIDENCE_THRESHOLD = 0.6`.
  Acceptance: `match_receipts` returns `None` for any filename not in `uploaded_files` and lists it in `missing`. `extract()` returns an `OCRResult` with a `confidence` between 0–1; a blank/white image returns confidence < 0.6; a clear receipt image populates at least one field.
  Verify: Test `match_receipts` in REPL with a dict of two filenames — pass three receipt_file values, one missing — confirm matched dict has None for the missing one and missing list contains its name. Test `extract()` with any receipt image file and print the returned OCRResult fields.

- [ ] **5. AI Category Classifier**
  Spec ref: `spec.md > Engine Layer > engine/ai.py`
  What to build: Implement `src/engine/ai.py` with `classify(category: str, ocr_raw_text: str) -> str | None`. Step 1 — restricted items fast-path: load `config/restricted_items.json`, check each item case-insensitively against `ocr_raw_text`; if any hit, return `"category_mismatch"` immediately without an API call. Step 2 — Groq API call: construct client with `Groq(api_key=st.secrets["GROQ_API_KEY"])`, call `client.chat.completions.create()` with exact system prompt from spec (`"You are an expense audit assistant..."`) and user prompt (`f"Category: {category}\nReceipt text: {ocr_raw_text}"`), `model="llama-3.3-70b-versatile"`, `max_tokens=10`, `temperature=0`. Step 3 — response validation: strip and lowercase; if not one of the three valid values, return `"category_unclear"`. Step 4 — error handling: wrap entire Groq call in `try/except`; on any exception return `None`.
  Acceptance: `classify("Office Supplies", "purchased wine and beer")` returns `"category_mismatch"` without calling the Groq API. A real Groq call with `category="Meals"` and restaurant receipt text returns `"category_consistent"`. Any exception returns `None`.
  Verify: Test the fast-path in REPL — call `classify()` with text containing "alcohol" and confirm immediate return without API call (add a print to verify). Then make a live Groq call with a real category and OCR text — confirm one of the three valid strings returned in under 3 seconds.

- [ ] **6. Rule Engine**
  Spec ref: `spec.md > Engine Layer > engine/rules.py`
  What to build: Implement all five rule functions in `src/engine/rules.py`. Each takes a row (`pd.Series`) and returns a `RuleResult`. (1) `check_amount_mismatch(row, ocr) -> RuleResult` — diff = `abs(row.amount - ocr.amount) / row.amount`; 0%: OK/NONE; 0–10%: UNCERTAIN/LOW; >10%: FLAGGED/HIGH. detailed_reason includes both amounts, calculated diff, and threshold. (2) `check_duplicate(row, df) -> RuleResult` — exact: same receipt_file OR same vendor+amount+date → FLAGGED/HIGH; fuzzy: same vendor + amount within 5% + date within 3 days → UNCERTAIN/MEDIUM; none: OK/NONE. Sets `duplicate_match_id`. (3) `check_missing_receipt(row, ocr) -> RuleResult` — found+readable: OK/NONE; found+low-confidence: UNCERTAIN/LOW; None (missing): FLAGGED/HIGH with `reason_summary = f"receipt file not found: {row.receipt_file}"`. (4) `check_category(row, ocr, ai_result) -> RuleResult` — loads `config/categories.json`; not in list: FLAGGED/MEDIUM; in list: map ai_result to status per spec; ai_result=None fallback: in list → UNCERTAIN/LOW with "AI-assisted check unavailable" note. (5) `check_suspicious_pattern(row, df, preliminary_results) -> RuleResult` — aggregates by employee_id; one weak signal: UNCERTAIN/LOW; multiple issues (≥2 of: repeated duplicates, repeated missing receipts, high FLAGGED count): FLAGGED/MEDIUM.
  Acceptance: `check_amount_mismatch` with row.amount=150, ocr.amount=120 → FLAGGED/HIGH, reason includes "+25%". `check_duplicate` with two rows sharing the same receipt_file → FLAGGED/HIGH. `check_missing_receipt` with ocr=None → FLAGGED/HIGH with filename in reason. `check_category` with category "Gambling" (not in approved list) → FLAGGED/MEDIUM.
  Verify: In Python REPL — construct test `pd.Series` rows and call each function. Test both passing and failing cases for each rule. Print the `RuleResult.status`, `severity`, and `reason_summary` and confirm they match expectations.

- [ ] **7. Orchestrator**
  Spec ref: `spec.md > Engine Layer > engine/orchestrator.py`
  What to build: Implement `src/engine/orchestrator.py` with `run_audit(df, receipt_map, groq_client) -> list[AuditResult]`. Per-row loop: call `ocr.extract(receipt_map[row.receipt_file])` → call `ai.classify()` only if ocr_result exists → run `check_amount_mismatch`, `check_duplicate`, `check_missing_receipt`, `check_category` → wrap all in `try/except`. After first pass, run `check_suspicious_pattern()` for all rows using preliminary results. `assemble(row, rule_results, ocr_result, ai_result) -> AuditResult`: highest severity wins for final status; collect all triggered rule names into `triggered_rules`; `reason_summary` from highest-severity rule; `detailed_reason` concatenated from all triggered rules. `error_result(row, error_msg) -> AuditResult`: FLAGGED/HIGH, reason = `f"processing error: {error_msg}"`. Update `st.session_state["processing_step"]` at each of the four pipeline stages.
  Acceptance: `run_audit()` on a 3-row DataFrame returns 3 `AuditResult` objects. An intentional error in one row (e.g. bad data type) produces FLAGGED result for that row without crashing others. `triggered_rules` lists all rule names that fired. `ai_assisted` field is True only when `ai.classify()` was called.
  Verify: Write a short Python script (no Streamlit) that builds a test DataFrame, calls `run_audit()`, and prints each result's `expense_id`, `audit_status`, `severity`, and `triggered_rules`. Confirm the output is correct for your test data.
